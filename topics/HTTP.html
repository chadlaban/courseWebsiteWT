Introduction

HTTP is one of the protocols used by the web or the internet. It is an asymmetric request-response client-server protocol, meaning, the client sends a request to a server, then the server, returns a response. HTTP is a pull protocol, meaning, the client pulls information from the server, instead of the server pushing information down to the client.

A qoute from the RFC2616: "The Hypertext Transfer Protocol (HTTP) is an application-level protocol for distributed, collaborative, hypermedia information systems. It is a generic, stateless, protocol which can be used for many tasks beyond its use for hypertext, such as name servers and distributed object management systems, through extension of its request methods, error codes and headers."

 

The Protocol

When you enter a URL in the address box (browser), it will translate the URL into a request message according to which specific protocol and sends that request to the server.

When the request reaches the server, these actions can take place:

  • It interprets the request, locates the request from a file under the server's directory, and it returns back the requested information to the client.

  • Interprets the request, locates the program kept inside the server, executes and returns the output to the client.

  • Returns an error message (unsatisfied request).

The browser will return thos response messages, it will interpret and display the contents of the message according to its

media type.Response headers: text/plain, text/html, image/gif, image/jpeg, audio/mpeg, video/mpeg, application/msword, and application/pdf

Response and Request Messages

An HTTP message contains a message header and a message body (optional).

HTTP Request Message

Request Line, this is the first line of the message header and it consists of:

  • request-method-name: this are request methods that are used by the client to send a request to the server, examples: GET, POST, OPTIONS and HEAD

  • request-URI: this is the resource requested.

  • HTTP-version: the versions used by the client.

Request Headers, the header is optional for the request message, but still, it consists of the following:

  • Host
  • Connection
  • Accept
  • Accept-Language

**note: this are just examples of what are the contents of a message request header.

Status Line, this is the first line in a response message, and it consists of the following:

  • HTTP-version: the HTTP version used in the session.

  • status-code: a 3-digit code generated by the server.

  • reason-phase: a short description of the status code.

Response Headers, this consists of the following:

  • Content-Type
  • Content-Length
  • Connection
  • Keep-Alive

**note: this are just examples of what are the contents of a message response header.

Response Status Code

This is found on the first line of a status code, generated by the server.

Status Code

A status code is a 3-Digit number:

  • 1xx - Informational
  • 2xx - Success
  • 3xx - Redirectional
  • 4xx - Client Error
  • 5xx - Server Error

Here are some of the commonly encountrered status codes:

  • 100 (Continue): Server received the request and is in the process of giving a response.

  • 200 (OK): Request is satisfied.

  • 301 (Moved Permanently): The resource being requested has been moved to a new location.

  • 302 (Found and Redirect/Moved Temporarily): it is the same as 301, but, it is just temporary.

  • 304 (Not Modified): In relation to the GET request, the server notifies the that there were no modifications made in the resource beign requested.

  • 400 (Bad Request): Server cannot interpret the request, this is caused by a syntax error.

  • 401 (Authentication Required): The resource beign requested is protected and requires credentials.

  • 403 (Forbidden): The Server declines the request of the client, even with valid credentials.

  • 404 (Not Found): The resource being requested cannot be located in the server.

  • 405 (Method Not Allowed): The server did not allow a method (PUT, POST, DELETE) for the resource being requested.

  • 408 (Request Timeout): The server took too long in responding in the usual time it must respond.

  • 500 (Internal Server Error): This is caused by server-side programming that is to respond to a request.

  • 502 (Bad Gateway): This due to a bad response to the upstream server.

  • 503 (Service Unavailable): This due to overloading/maintainenance.

  • 504 (Gateway Timeout): The gateway has received a timeout from the upstream server.
